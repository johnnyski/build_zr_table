build_gauge_db: 
Build Rain Gauge Database
Updated: May 7, 1998.

Synopsis

    build_gauge_db  [-v] [-f gauge_gdbm_file] [-d infile_modification_date_range] 
     input_list 

Description

This program creates/updates rain gauge database in GDBM format to be used by merge_radarNgauge_data. build_gauge_db accepts rain gauge files and/or directories of rain gauge files as inputs, where rain gauge files are 2A-56 products from either a single or multliple gauge sites. (Note: Since the database query time increases as the size of the database increases, it is better to construct one gauge database for each radar site instead of one gauge database for multiple radar sites to improve the performance of the radar and gauge merging processing). Since rain gauge files may be modified after the gauge database has been created, this program allows multiple executions on the same input rain gauge file(s). When execution on the same rain gauge file(s) again, the program adds new entri(es) into the existing database but updates the existing entri(es) as appropriate. The program for querying the gauge database is called query_gauge_db; it is used to query what's in the gauge database. In addition, the program for validating the gauge database is called validate_gauge_db.

Options

-v Show verbose messages of program execution.

-f Specify filename for the output database. Default is $GVS_DB_PATH/gauge.gdbm. Default of $GVS_DB_PATH is /usr/local/trmm/GVBOX/data/db.

-d Specify the file modification date range. The file modification date is the date stamp when the file was last modified; it is not the date appeared on the input filename(s) (Note: Type 'ls -l  2A56*' to see file modification dates for the gauge_files).  Only add gauge files that has the file modification date within the specified date range. Syntax: mm/dd/yy[yy]- | -mm/dd/yy[yy] | mm/dd/yy[yy]-mm/dd/yy[yy]. Default: Add all gauge fileslisted or from the specified directori(es). For example,

  • Specify '-d 1/23/98-' to add only gauge files having the file modification date of 1/23/98 or later.
  • Specify '-d -2/23/98' to add only gauge files having the file modification date earlier and/or equal to 2/23/98.
  • Specify '-d 1/23/98-2/23/98' to add only gauge files having the file modification date between 1/23/98 and 2/23/98 inclusive.

Inputs

A list of rain gauge file(s) and/or directori(es) contained rain gauge files separated by space. Rain gauge files are 2A-56 products.

Note:

  • To build a gauge database for a single radar site, specify the gauge files and/or directories of that particular radar site only.
  • To build a gauge database for multiple radar sites, specify the gauge files and/or directories of multiple radar sites.
  • Since the database query time increases as the size of the database increases, it is better to construct one gauge database for each radar site instead of one gauge database for multiple radar sites to improve the performance of merge_radarNgauge_data.

Temporary Files or Directories

None.

Output

Rain gauge database in GDBM format. The database may contain the gauge rain rates for either a single or multiple radar sites depending on the input sources. This file will be updated if it already exists . Note: The size of the gauge database may be very large.

Return Value

build_gauge_db returns 0 upon successful completion; 2 upon interruption; non-zero, otherwise.

build_pmm_zr_table: 
Match Z/R/Classification Pairs
Updated: December 23, 1998.

Synopsis

    build_pmm_zr_tables [-v] [-c] [-h height] [-t temporal_gauge_window_duration] [-o offset_temporal_gauge_window] 
                        [-r r1,r2,...] [-d] infile outfile

Description

This program is the final processing step for constructing ZR table(s). It matches Z/R/classification pairs from the input file which is the result of merge_radarNgauge_data. It produces ZR tables based on: 1) Rain classification at the raingauge and 2) Range of the raingauge from the radar site. For example, given two rain types (convective & stratiform) and three range intervals ([0,50], [50,100], [100,150]), then six (3x2) distinct ZR tables will be produced. All ZR tables will be stored in one file.

Options

-v: verbose execution.
-c: Specify this flag if  want to use only the cell over gauge from spatial gauge window. Note: A cell has area 4km^2 (2km x 2km) (Default: use 9(=3x3) cells from one height)
-h: Specify height at either 1.5 or 3.0: Use window cells from specified height(km) from 2A-55. (Default: range-dependant)
-t: temporal gauge window interval in minutes. (Default: 6) .
-o: Offset temporal gauge window from the input file's temporal window center in minutes. (Default: 0).
-r r1,r2,.... Specifies desired range intervals in KM for ZR tables where r1, r2,..., rN are: [0,r1], [r1,r2], ..., [r(N-1),150]. rN <= 150.
-d: Disregard rain classification. Lump all classes together.

Input

infile is the merged radar gauge file produced by merge_radarNgauge_data..  Note: The merged radar gauge file should be qc'ed before applying to this program.

Temporary Files or Directories

None.

Output

outfile will contain the produced ZR table(s). Multiple ZR tables are possible; see option -r above.

Return Value

build_pmm_zr_table returns 0 upon successful completion; 2 upon interruption; non-zero, otherwise.

get_radar_data_over_gauge: 
Extract Radar Windows and Convective/Stratiform Classifications
Updated: April 21, 1998.

Synopsis

    get_radar_data_over_gauge  [-v] [-S site_name] [-x gauge_win_xmax] [-y gauge_win_ymax] 
          [-c rain_class_type] [-g gauge_locations_top_dir]
          2A-54_granule_hdf 2A-55_granule_hdf first_zr_intermediate_outfile

Description

This program corresponds to the first step for constructing ZR tables. It extracts radar windows and convective/stratiform classifications using two carpis high. If rain classification type is conv/strat, then the 2A-54 convective/stratiform classification will be used. It creates a first ZR intermediate file which will be used by the radar/gauge merging program, merge_radarNgauge_data

Options

-S Specify site name. Default: get from 2A-55 file.
-x Specify the X length of the gauge window in km. Default: 4.0
-y Specify the Y length of the gauge window in km. Default: 4.0
-c Specify rain classifications number (Uniform; conv/strat). Default: conv/strat
-g Top directory of gauge site locations data. Default: $GVS_DATA_PATH. The site locations data files are actually in a subdirectory called sitelist but don't specify that here.  Specify the directory where 'sitelist/' resides. The default value for $GVS_DATA_PATH is /usr/local/trmm/GVBOX/data.

Inputs

  • 2A-54_granule_hdf Cartesian_CS_map for one granule in HDF.
  • 2A-55_granule_hdf 3-D reflectivities for one granule in HDF.
  • Gauge site locations information. See option -g in the Options section.

Temporary Files or Directories

None.

Output

first_zr_intermediate_outfile This file will be appended or created if it does not exist. It contains radar data over gauges along with rain types.

Return Value

get_radar_data_over_gauge returns 0 upon successful completion; 2 upon interruption; non-zero, otherwise.

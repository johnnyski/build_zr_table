build_zr_table Generate the ZR table from command line parameters for Stratiform and Convective raintypes.
Updated: February 8, 1999.

Synopsis

    build_zr_table [--site {KMLB, KWAJ, ...}]
                   [--sdate mm/dd/yyyy]
                   [--stime hh:mm:ss]
                   [--edate mm/dd/yyyy]
                   [--etime hh:mm:ss]
                   [-k x1 [-k x2 ...]]
                   [-a y1 [-a y2 ...]]
                   [-t t1 [-t t2 ...]]

                   [-h | --help]
 

Note:
          -k, -a, and -t should be entered together.  And the number
           of parameters should match.  (xn, yn, tn) defines a triplet
           of (k, a, and raintype) that specify the function: R=kZ^a.

           Currently, specify both x1 and x2, or y1 and y2 or t1 and t2.
           Currently, -t is not implemented, but, the rain types Stratiform
                      and Convective are known (hard coded) and the order
                      shown as the default is accurate: t1 = Stratiform
                      and t2 = Convective.

           Soon to be implemented:
               -k x1 -k x2 ... -k xn
               -a y1 -a y2 ... -a yn
               -t t1 -t t2 ... -t tn

Description

Generates a dual ZR table using command line parameters.   It generates a ZR table for the rain types: Stratiform and Convective.   The rain types cannot be changed.  The ZR table contains a range dependancy.  That is to say that the ZR table has meaningful values for specific ranges of the data.  Currently, the ranges are: 0-15km, 15-100km and beyond 100km.   These cannot be change.  The ZR table contains meaningful values for the 15-100km range only.  The other ranges are flagged as MISSING (set to -99).

The -site parameter, by default, uses the file /usr/local/trmm/GVBOX/data/gv_radar_site_info.data to determine the latitude and longitude of the radar site.  Diagnostic messages are produced, if the site cannot be found or the gv_radar_site_info.data file cannot be found.

          (See /usr/local/trmm/GVBOX/data/gv_radar_site_info.data)
--sdate mm/dd/yyyy]  -> Start date. 00/00/0000
--stime hh:mm:ss]    -> Start time. 00:00:00
--edate mm/dd/yyyy]  -> End   date. 00/00/0000
--etime hh:mm:ss]    -> End   time. 00:00:00
-k x1 [-k x2 ...]]   -> Default: -k 300.0 -k 300.0
-a y1 [-a y2 ...]]   -> Default: -a 1.4 -a 1.4
-t t1 [-t t2 ...]]   -> Default: -t Stratiform -t Convective

-h | --help]       -> HELP.  Print this usage.

Input

None.  The program is completely command line driven.

Temporary Files or Directories

None.

Output

The ZR table is written to STDOUT.  The output is lengthy.

Return Value

None.
